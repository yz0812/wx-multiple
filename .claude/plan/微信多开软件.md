## 📋 实施计划：微信多开软件

### 任务类型
- [x] 前端
- [ ] 后端
- [x] 全栈

### 技术方案
基于全新 Tauri 2 + React + TypeScript + Vite + Tailwind + shadcn/ui 项目实现一个本地桌面工具。前端提供“微信目录”“启动数量”“开始启动”三个核心交互；Tauri 后端负责扫描常见微信安装路径，并在用户点击开始后直接按 `微信双开.bat` 的逻辑用 Rust 启动 `Weixin.exe` 多次。默认不依赖外部 bat 文件，避免打包后还要额外挂脚本。

增强后的结构化需求：
- 目标：做一个 Windows 本地桌面工具，用于选择/确认微信安装目录，并按指定数量启动微信客户端。
- 技术约束：Tauri 2、Rust、React、TypeScript、Vite、Tailwind CSS，UI 使用 shadcn/ui。
- 范围边界：仅实现本地目录扫描、手动填写/选择目录、数量选择、启动执行、基础成功/失败提示；不包含托盘、开机启动、配置持久化、多语言、自动更新。
- 自动扫描规则：应用启动时扫描 Windows 常见安装路径，若找到 `Weixin.exe` 则自动回填；找不到时允许用户手动输入或选择目录。
- 启动规则：Tauri 后端接收 exe 完整路径和数量参数，循环启动进程，每次间隔约 500ms。
- 验收标准：应用启动后能自动识别常见微信目录；用户可手动改目录；可选择启动数量；点击开始后按数量成功拉起微信；路径无效时有明确报错。

### 实施步骤
1. 初始化 Tauri 2 项目骨架 - 产出 React + TypeScript + Vite + Rust 的基础桌面工程。
2. 集成 Tailwind 和 shadcn/ui 基础组件 - 产出可复用的输入框、按钮、选择器等 UI 基础。
3. 实现前端主页面 - 产出包含目录输入、目录选择、数量选择、开始按钮、状态提示的单页界面。
4. 实现 Tauri 扫描命令 - 产出扫描常见微信安装路径并返回命中结果的 Rust 命令。
5. 实现 Tauri 启动命令 - 产出按数量循环启动 `Weixin.exe`、每次间隔 500ms 的 Rust 命令。
6. 连接前后端调用链 - 产出页面初始化自动扫描、点击按钮调用启动命令的完整流程。
7. 补充基础校验与验证 - 产出路径存在性校验、错误提示，并完成本地联调验证。

### 风险与缓解
| 风险 | 缓解措施 |
|------|----------|
| 用户机器上的微信安装路径不在常见目录中 | 保留手动输入和目录选择，自动扫描只做默认填充 |
| 启动次数过大导致用户误触发大量实例 | 数量控件限制合理上限，例如 2-10 |
| 直接启动 exe 失败 | 在后端先校验路径是否存在且文件名为 `Weixin.exe`，失败时返回明确错误 |
| 不同机器路径含空格或中文 | Rust 直接使用进程 API 传路径，不拼 shell 字符串 |
| Tauri 2 初始化细节与实际 CLI 模板有差异 | 实施时按当下官方模板落地，不预设不存在的脚手架细节 |

### 验证方式
1. 启动应用，确认页面首次加载会自动扫描常见微信目录。
2. 人工修改为正确目录，确认界面允许手动输入/选择。
3. 选择 2、3、4 等不同数量，点击开始，确认微信按数量被拉起。
4. 输入不存在目录，确认页面得到清晰报错而不是静默失败。
5. 打包后在目标机器验证，确认无需依赖外部 bat 文件也能正常运行。
